网址就是一个字符串，由协议，主机，端口，路径组成

https://www.baidu.com

主机

域名，有一个机构为这些数字（IP）取名字，通过DNS服务器来告诉你的浏览器他们之间的对应关系

利用ping命令可以找到域名对应的IP地址

端口

定义：IP相当于地址，端口是收信人姓名（一个地址比如公司、家只有一个地址，但是却可能有很多收信人）

端口就是一个标记收信人的 数字

端口是一个16位的数字，所以范围是 0 ~ 65535

http协议

定义：一个传输协议，协议就是双方都遵守的规范

为什么叫超文本传输协议呢，因为收发的是文本信息

  1. 浏览器（客户端）按照 规定的格式 发送文本数据（请求）到服务器
  2. 服务器 解析 请求，按照 规定的格式 返回文本数据（未必全是文本数据）到浏览器
  3. 浏览器 解析 得到的数据，并做相应处理

请求和返回是一样的数据格式，分为4部分：

  1. 请求行或者响应行
  2. Header（请求的 Header 中 Host 字段是必须的，其他的都是可选（协议宽松））
  3. \r\n\r\n（连续两个换行回车符，用来分隔 Header 和 Body）
  4. Body（可选）

请求的格式，注意大小写（这是一个不包含Body的请求）

'GET / HTTP/1.1\r\nhost:g.cn\r\n\r\n'

打印出来如下： ''' GET / HTTP/1.1 Host:g.cn '''

其中，

  1. GET是请求方法（还有POST等，这就是个标志字符串而已）
  2. / 是请求的路径（这代表根路径）
  3. HTTP/1.1 中，1.1是版本号，通用了快20年了
